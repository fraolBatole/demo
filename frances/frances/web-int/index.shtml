Take a quick survey


If you find any problems, would like to see any features added, or just have general comments, please take our survey or email us at sondag@cs.iastate.edu. We are very interested to hear who is using our tool and in what ways. This will help us develop future versions of this tool.

Features currently under construction

is an active project. If there are any features you would like to see added, or if you find any bugs, please notify us. In the future, we plan to release the source code as an open source project. For now, please continue to use the web interface to the tool. Our current work and future plans for this tool are the following:

  • Multiple language support
    Details: Currently, the tool only supports C. Add support for C++, Fortran 77, Fortran 90, Java, Python, etc.
    Difficulty: >Easy
    Status: >In progress
    Progress: C++ and Fortran 90 work. Fortran 90 still needs some fine tuning. Also, further testing is needed for C++. Other languages have not yet been implmented.
  • Target assembly language
    Details: Currently, the tool uses x86 AT&T syntax. Add support for MIPS and x86 Intel syntax.
    Difficulty: >Easy
    Status: >Completed
    Progress: Currently, the tool uses x86 AT&T syntax. Support for MIPS and x86 Intel syntax is now available experimentally.
  • Custom transofmrations
    Details: Add support for custom (user supplied) high level and low level langauge combinations.
    Difficulty: >Intermediate
    Status: >No progress
    Progress: Currently, the tool uses standard high level languages and standard low level languages. Support for custom languages is desired so the tool can be closely used with existing course langauges.
  • Program tracing
    Details: Trace an execution's behavior on the system with virtual stack, heap, registers, etc.
    Difficulty: >Difficult
    Status: >Completed
    Progress: Initial support as part of the Frances-A project.
  • Hot-spot finder
    Details: Find hot-spots in code via dynamic profile and static analysis where possible.
    Difficulty: >Intermediate
    Status: >No progress
  • Analysis/Optimizations
    Details: Illustration for common static analysis and optimizations.
    Difficulty: >Difficult
    Status: >No progress
    Progress: Static analysis techniques are coded but need to be interfaced with interface. Optimizations need to be coded and interfaced. >Difficult
    Status: >No progress
