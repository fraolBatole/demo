Overview

This programming guide describes the Ptolemy language and the tools which are part of the Ptolemy development environment.

If you are completely new to aspect-oriented software development, you should first read about Implicit Invocation (II) design style, then about aspect-oriented languages.

If you are completely new to Ptolemy, you should first read Getting Started with Ptolemy for a broad overview of programming in Ptolemy. If you are already familiar with Ptolemy, but want a deeper understanding, you should read our ECOOP 2008 paper. If you want the complete formal definition of Ptolemy, you should read our technical report on Ptolemy.
